# 簡易 SCM 系統的 PRD
了解，下面是針對「SCM（供應鏈管理系統）」功能的產品需求文檔（PRD）草案，專為用 Python Flask 實作全站 DEMO 且不使用 API 或資料庫設計。

***

## 產品需求文檔 (PRD) — SCM 供應鏈管理系統 DEMO 網站

### 產品概述
本系統為供應鏈管理（SCM）功能的示範網站，採用 Python Flask 全站架構開發，用於展示優化供應商、物流、庫存及配送等關鍵供應鏈環節功能，不使用 API 或資料庫，資料全部以後端記憶體或靜態檔案暫存。

### 目標用戶
- SCM 學習者、系統設計演示需求者
- 供應鏈及物流管理相關職務新人培訓

### 主要功能

#### 1. 供應商管理
- 顯示供應商列表（靜態初始數據）
- 新增、編輯、刪除供應商（操作記錄暫存在後端記憶體）
- 供應商基本資訊：名稱、聯絡人、電話、地址、供應商品項

#### 2. 物流管理
- 顯示運輸路徑及車輛狀態（使用靜態資料模擬）
- 新增物流訂單（內存保存）
- 物流訂單追蹤狀態（待出貨、運輸中、已送達）

#### 3. 庫存管理
- 顯示庫存商品列表及庫存數量
- 商品入庫與出庫模擬（即時更新記憶體資料）
- 庫存警示（低於臨界值提示）

#### 4. 配送管理
- 配送訂單建立與管理
- 配送狀態更新（準備、配送中、完成）
- 配送路徑與訂單配對展示

### 非功能需求
- 使用 Python Flask 以伺服器端模板渲染（Jinja2）
- 不使用 API，不連接資料庫；所有變動均保存在伺服器記憶體，重啟即重置
- 簡潔直觀的前端介面，使用 Bootstrap 或基本 CSS
- 頁面響應式布局，適合桌面及平板瀏覽

### 技術架構
- Python Flask (後端伺服器與模板渲染)
- HTML + CSS (Bootstrap 可選用)
- 資料結構使用 Python dict/list 存放供應商、物流、庫存、配送資料
- 伺服器記憶體中臨時存放操作資料，不持久化

### 介面設計（粗略示意）
- 頁首選單：供應商管理 / 物流管理 / 庫存管理 / 配送管理
- 各功能頁含列表展示與表單操作
- 每個功能模擬 CRUD（建立、編輯、刪除）操作即時更新記憶體物件

### 範圍限制
- 不納入多用戶權限管理
- 不透過前後端分離架構，純伺服器渲染
- 不保存或讀取資料庫，網站重載資料恢復到靜態初始
